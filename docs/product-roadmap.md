## 二、P0 Bug 列表（必须先修）

这些 bug 影响核心链路正常运行，优先级最高：

| # | Bug | 严重性 | 修复方案 |
|---|-----|--------|---------|
| B1 | **WXML 函数调用不生效** — `isSelected()`、`difficultyLabel()`、`isFlavorSelected()` 在模板中直接调用 Page 方法，微信小程序不支持 | 🔴 高 | 改为 `data-*` + WXS 或在 setData 时预计算选中态字段 |
| B2 | **云开发 init 重复冲突** — `app.js` 写死真实 env，`index.js` 又 init `your-env-id`，重复初始化 | 🔴 高 | 删除 index.js 中的二次 init，统一在 app.js |
| B3 | **降级菜谱"换一批"不排除** — 本地 3 道固定菜不根据 exclude 过滤 | 🟡 中 | 本地降级时过滤 excludeNames，或扩大降级菜谱库到 10 道 |

## 三、路线图：Now → Next → Later

### 🟢 Now（本周完成 — P0 修 bug + 对接云函数）

目标：**核心链路可用，手动选食材 → LLM 推荐 → 步骤引导 全流程跑通**

| # | 任务 | 预估 | 备注 |
|---|------|------|------|
| N1 | 修复 WXML 函数调用 bug | 1h | isSelected / difficultyLabel / isFlavorSelected 改为 setData 预计算 |
| N2 | 删除 index.js 重复 cloud.init | 15min | 统一走 app.js 全局初始化 |
| N3 | 降级菜谱 exclude 过滤 | 30min | 扩充本地菜谱库到 8-10 道，换一批时排除 |
| N4 | LLM 调用切回云函数优先 | 1h | llm.js 改为先 `wx.cloud.callFunction` → 失败再客户端直连 → 最终本地降级 |
| N5 | 云函数部署上线 | 30min | npm install + 微信开发者工具上传 + 配置环境变量 |

**Now 完成后的验收标准**：手动选食材 → 菜谱推荐 → 点一道菜 → 步骤引导，全链路流畅无 bug。
